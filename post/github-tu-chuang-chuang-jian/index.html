    Github图床创建

    2020-02-15 | 技术 markdown github 图床 | 3 min read

    此文章发表于 673 天前，请注意文章时效

    今天给妹妹写转租的广告，因为我自己手机自带的笔记软件不能导出图片，也不想使用截图，所以就在纯纯写作上写了转租广告。最后在加上我自己的微信二维码，结果在加图片的时候出了一个问题，纯纯写作不支持本地图片，只能使用线上图片。

    虽然后面使用MarkdownX解决了问题，但是在Markdown中使用线上图片的问题总得解决一下（我之前都是直接使用本地图片，电脑上的也是这样。。。）

    Markdown的图片使用在线插入有一些好处：

    • 不用担心文件迁移时，造成图片路径变动，结果文件中的图片不可见
    • 比较好的统一管理图片

    当然也有不少不方便的地方，比如：找到一个靠谱的图床供应商。相信你也见过不少人的博客里的图片都不可见，或者变成一些禁止外链的说明。

    现有方案

    目前有很多的图床，有收费的有免费的。

    • 七牛云【需要自己备案域名，临时测试域名一段时间会收回】
    • SM.MS 【担心垮掉】
    • 微博【可以被反查到发布者参见此处】
    • 又拍
    • 腾讯 【备案】
    • 阿里

    这里有几个考量：

    1. 稳定。 不能出现像上面说的，因为图床垮了，结果图片都没有了。
    2. 速度。要能提供足够快速的访问速度，防止出现图片刷半天才出来。
    3. 方便。方便易用，不用进行备案。

    已经有的车轮子

    考虑来考虑去，结果看到有人可以用Github来当作图床，这就方便多了。有很多文章都报道了如何来搭建，本着不重复造轮子的原则，我将他们的链接放这里：

    1. 20200108 - PicGo + GitHub 搭建个人图床工具
    2. 如何在自己的GitHub搭建图床

    问题解决

    上传失败，服务端出错，请重试

    在安装好picGO并且配置了自己的仓库后，并没有一帆风顺的上传图片，而是出现了上传失败，服务端出错，请重试这样的问题。出现这个问题，大概率是因为自己的github配置出错了。这时要回去检查下，自己的用户名，仓库名有没有空格。如果有空格需要用“-”来代替。并且[username]/[repository]“/”两边也不能有空格。

    本文链接：https://willisfusu.github.io/post/github-tu-chuang-chuang-jian/
