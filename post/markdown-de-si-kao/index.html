    Markdown 的思考

    2018-03-28 | 笔记 markdown 想法 | 6 min read

    此文章发表于 1366 天前，请注意文章时效

    对于Markdown或者LaTex之类的工具可以提高输入效率，我是一直有疑问的。

    对于 Word 与 Markdown 之间的差别，或者说 Markdown 的优势，不少推荐者都只是提到一点：提高输入效率。那么是如何提高输入效率的呢？他们说，是因为输入者将注意力放到的文字本身上，没有像 word 一样去关注文字的样式。这个就是我特别疑问的一点。

    可以直观理解的一点是，任何的文字输出，只要最终是含有样式的结果，那么必然是经过两次创造的。第一次创造是在脑海中完成的，是想像出来的含有样式的结果。第二次是使用各种工具按照脑海中的样本来进行按图施工。所以在具体的施工过程就有两种不同和方案：

    • 在施工过程中，边施工，边完成样式
    • 施工完成后，统一装修

    说到这里就可以理解了，我们将现在有的两种文本处理工具对应到这两个方案上：对比我们可以看到，使用 Markdown 的用户，在输入的过程中，需要完成的活动是：脑中创造样式结果同时手上输入文字。Word 用户需要完成的活动是：输入文字–脑中创造样式–完成样式。可见，Markdown 并没有像很多人说的那样，不去关注样式，恰恰相反，Markdown 用户，在输入的过程中，精力更加分散，因为需要脑海中不断的创造样式，并且输入这样样式的标记符号。 相比之下 Word 用户，可以从容在只专注于文字，在文字完成之后，再去赋予文字样式。

    • 在施工过程中，边施工，边完成样式rightarrow\\rightarrowrightarrow Word & Markdown
    • 施工完成后，统一装修 rightarrow\\rightarrowrightarrow Word

    结论很明显：Markdown 专注于文字是一个伪命题

    至于在完成样式的方便程度、样式的可定制程度、样式的美观程度的对比，想必大家心里也有一个数：Markdown 这种轻量级的完全不是 Word 的对手。

    那么 Markdown 有没有提高输入效率？

    虽然上文说专注于文字是一个伪命题，但是这里的答案却是提高了。

    为什么？

    这是因为 Markdown 满足的一种特定的需求：极其简单的样式、相对好看的排版

    前文说到了，使用 Markdown 其实是分散了大脑的注意力，增加了大脑的负荷，那么为什么还会提高输入效率呢？其实只要将增加的负荷、分散的注意力降低到一个很低，并且可以接受的程度下，那么总的结果呈现所需要的时间，就完全可以低于 Word 用户。那么怎么样来降低负荷呢？使用最简洁的样式，最少的特殊符号。说白了，就是 限制需求。

    大家可以想一想，在使用 Markdown 的过程中，你使用最多的是什么样式，我使用最多的就是下面几个：

    • 标题 #
    • 列表 * - 1. []
    • 加粗* _斜体_
    • 链接 [] () \ \
    • 输入数学公式E=mc2E=mc^2E=mc2

    这几个样式，都是输入简单，并且最终得到的结果还是相对好看的。当最终的结果只是简单的样式，相对好看的排版的话，使用 Markdown 的确是提高了效率，并且，在有的电脑上，打开 Word 软件也是需要不少时间的:smile

    说到这里，不少人可能会问，为啥那么多人会说 Markdown 比 Word 的效率更高呢？不排除几个原因：

    1. 装
    2. 电脑上没有 Word，或者不理解且不会正确的使用 word
    3. 有特定的需求，但是在说效率高这个结论的时候，并没有把自己需求的前提说明出来

    Markdown 可以取代其它工具成为主力文字编辑器么？

    这个需要看需求，像上面说到的，如果一个人的需求就是极其简单的样式，相对好看的排版，那么，我觉得 Markdown 完全可以取代 Word，并且在这种需求下，我也是这样做的（简单的列表、不需要富文本的样式文章）。至于其它在网上看到的，使用 Markdown 来写学术论文、来画图，我想，还是一笑了之吧。

    且不说在文件的交流上可能带来的麻烦，就画图以及排版的可定制化程度之低，就足够用户喝一壶了。并且，我个人认为，文件交流上带来的困难更大。试想如下场景就可以了：

    • 论文书写过程中，参考文献的插入，修改
    • 论文修改过程中，老板的批注，md 应该是不能便捷的做批注的
    • 论文写完之后需要投稿，可是据我投稿期刊的经验来说，国内，国外没有哪个主流期刊接收 md 文件。
    • 画图之后，美化如何来做？

    Markdown 并不是一个全能的工具，以上所说的需求，都已经有成熟且为大众接受的解决方案，试图在 markdown 上完成这样的需求，只能是 徒劳无功。

    那么 Markdown 可以用来做什么呢？

    其它人怎么折腾我不晓得，我个人的用法就是像我上面所说的那样，在需要极其简洁的样式时，并且还需要有一个好看的排版，我就会使用 markdown。
    比如：

    1. 网站博客文章，配合 Hexo 很好用
    2. 读学术文章过程中，需要记录一些突发奇想的点子，疑问。
    3. 大部分 markdown 的编辑器，可以用来打开一些代码文件之类的，有代码高亮，并且不会乱码。。。。

    总的来说，就是代替了 windows 自带的文本编辑器。
